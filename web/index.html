By combining deep static analysis with behavioral heuristics, we create a fortified perimeter around your development environment. We don't just check for known vulnerabilities; we look for intent.

Every SafeDev command, mission-ready

Pulled from the README and current CLI implementation, these examples match the SafeDev workflow from first install to repo inspection, dependency triage, and dashboard launch.

terminal

Install & Upgrade Operations

safedev install requests

Scan then install a pip package only after risk review.

safedev install lodash --ecosystem npm

Use the same pre-trust workflow for npm packages.

safedev upgrade requests

Upgrade dependencies only after scanning the latest target.

safedev uninstall flask --yes

Remove packages cleanly without extra prompts.

safedev install-deps requirements.txt --yes

Install only dependency-file entries that stay under your threshold.

search_insights

Scan & Analysis Operations

safedev clone https://github.com/user/repo

Scan a repository in isolation before it enters your workspace.

safedev scan ./my-project

Inspect a local folder for suspicious rules, files, and patterns.

safedev scan ./my-project --output json

Export machine-readable scan data for automation or CI.

safedev scan-deps requirements.txt --threshold 3

Review every listed dependency with a stricter blocking policy.

safedev check --ecosystem npm

Use SafeDev as a single entry point for dependency health checks.

safedev audit

Trigger npm security audit from the SafeDev surface.

dashboard

Metadata, Runtime, and Dashboard Commands

safedev list

List installed packages in the active ecosystem.

safedev list --outdated

Surface dependencies that are lagging behind.

safedev show requests

Inspect metadata for an installed package.

safedev freeze

Export your Python environment as a requirements snapshot.

safedev version

Print SafeDev, Python, pip, npm, node, and git versions.

safedev run build

Run npm scripts through one trusted CLI surface.

safedev ui

Launch the Streamlit dashboard for visual scanning, history, exports, and command discovery.

tune

Core Flags

--ecosystem pip|npm

Switch between Python and JavaScript package ecosystems.

--threshold N

Tighten or relax the blocking policy for scans and dependency review.

--force

Bypass confirmation and continue when you explicitly accept the risk.

--output text|json

Choose between human-readable reports and automation-friendly output.
